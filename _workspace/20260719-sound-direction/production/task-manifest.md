1. `D0 → D1` freezes the sound-only contract. `DS1 ∥ AE1 ∥ E1 ∥ QA1 ∥ PM1` may inspect independently; none may claim a runtime or gate result.
2. `DS1 + AE1 + E1 + QA1 → D2`. Stage 1 exits only when the sound identity, current-file inventory, runtime boundary, and focused evidence plan exist; missing evidence is `FIX`.
3. `D2 → (DS3 ∥ DS4 ∥ AE2)`. Script parity and escalation direction precede any candidate approval; local generation is preferred and provider credentials are not assumed.
4. `(DS3 + DS4 + AE2) → E2 → E3`. Runtime integration consumes only approved candidates with actual files and provenance. Every new path must close runtime, manifest, cache, and Pages surfaces together.
5. `(E2 + E3) → (QA2 ∥ QA3 ∥ PM2) → D3`. QA measures cue behavior, parity, accessibility, and diversity; PM confirms no commercial/fairness coupling; director arbitrates conflicts numerically.
6. `D3 PASS → (E4 ∥ E5 ∥ QA4) → D4`. Stage 3 cannot pass from static inspection alone; offline, accessibility, renderer/input, lifecycle, and deployment evidence must be recorded.
